---

You are an expert in education operations, volunteer-run learning systems, mentoring programs, and academic acceleration models.

Create a detailed operating manual for the Gurudwara-based College Prep / Academic Excellence Program (CCP) described in the configuration and blueprint above.

This manual is for real volunteers. It must explain exactly how the program operates day to day and week to week. It should operationalize the blueprint — not re-argue its design principles.

**OVERARCHING DESIGN REQUIREMENT:**
Write for the minimum viable pilot first. Every section should describe what volunteers actually do in the first 90 days, then note how it evolves as the program matures. Do not design for the ideal state — design for the launchable state.

**OPERATING PRINCIPLES:**
1. Flipped classroom compliance is the most critical daily requirement. Every role playbook must address it explicitly.
2. Mentors verify Google Classroom completion daily using a simple check question provided by the tutor — not by checking login data.
3. Content curation follows a defined weekly workflow. It is not an improvised tutor responsibility.
4. All real-time communication runs through four defined WhatsApp groups. Norms for each group must be explicit.
5. The program must function when parents are minimally available.
6. Plain language throughout. This will be read by volunteers, not education professionals.
7. Parent-facing communication should be designed in plain English and Punjabi where appropriate for accessibility.

**1. Program Overview**
- Mission and philosophy
- What the program is and what it is not
- Desired outcomes by grade band (7–8 Foundation, 9–10 Acceleration, 11 Capstone)

**2. Organizational Structure**
Practical org model for the pilot. Include role of each persona, reporting lines, WhatsApp group assignment, communication flow, and decision-making authority.

**3. Role Playbooks**
For each persona — Program Lead, Content Curator, Tutor, Mentor, Parent, Student — provide:
- Purpose in one sentence
- Detailed responsibilities for pilot phase
- Time commitment (be specific and realistic)
- Daily and weekly checklist
- WhatsApp group(s) they belong to and their communication norms
- Success measures
- Motivation and retention approach

**4. Tutor Playbook**
Detailed instructions for tutors:
- How to run a flipped classroom session: comprehension check first, not lecture first
- Pre-session verification: how to check Google Classroom completion and what to do if students have not prepared
- Protocol when multiple students have not completed pre-work
- Session planning for mixed-ability groups within a single grade band
- Math-specific session design for pilot phase
- How to introduce PSAT-style reasoning progressively
- How to identify and support struggling students without derailing the group
- How to enable peer teaching within the 75-minute session
- Coordinating with mentors via WhatsApp Group 2
- Giving parents simple updates via WhatsApp Group 3

Included templates:
- Sample 75-minute Math session plan showing flipped classroom structure
- Tutor weekly checklist
- Tutor monthly review checklist
- Non-compliance protocol: what to do when students arrive unprepared

**5. Mentor Playbook**
Detailed instructions for mentors:
- The mentor's exact role: accountability partner, not teacher
- How to run the daily 10–15 minute check-in
- How to verify Google Classroom completion without being a content expert (use tutor-provided check question)
- Sample scripts: encouragement, accountability, redirection, escalation
- How to communicate via WhatsApp Group 2 (mentor-tutor) and direct message to parent
- What to do when a student claims content is too hard
- What to do when a student is disengaged for multiple days
- How and when to escalate to the tutor
- What mentors must never do

Included templates:
- Sample daily mentor check-in routine (minute by minute)
- Weekly mentor report template for WhatsApp Group 2
- Mentor do's and don'ts (one page)
- Mentor motivation and recognition system

**6. Parent Playbook**
Simple guide for parents:
- How to support learning without having to teach
- How to create a consistent home routine around the 30–45 minute daily practice
- How to use WhatsApp Group 3 effectively
- How to read their child's Google Classroom progress
- What to do when they are too busy to supervise
- How to work with the mentor
- Parent communication needs in both English and Punjabi

Included:
- Parent expectations (one page, plain language)
- Parent communication rhythm via WhatsApp Group 3
- Parent FAQ
- A Punjabi version of the parent playbook suitable for community distribution
- Guidance on which communications should always be bilingual (English + Punjabi)
- WhatsApp Group 3 message templates in both English and Punjabi
  for the most common situations: weekly update, missed practice
  alert, session reminder, milestone celebration

**7. Student Playbook**
- Daily practice expectations and Google Classroom workflow
- How to navigate the weekly Google Classroom structure
- How to use KA and test-prep materials as assigned
- How to ask for help from mentor and tutor
- How students grow from learner to peer helper to future mentor

Included:
- Student motivation and recognition system
- Recognition milestones
- Sample weekly student checklist

**8. Student Progress Engine**
The exact operating system:
- Daily 30–45 minute practice model via Google Classroom
- Weekly targets for Math pilot
- Mentor accountability check tied to Google Classroom completion
- Pre-session tutor verification
- Escalation ladder: day 1–2 missed → mentor encourages;
  day 3 missed → mentor flags tutor via WhatsApp Group 2;
  week 2 of non-engagement → program lead review
- Attendance thresholds and missed-session protocol as
  defined in configuration
- Minimum viable practice protocol for days when technology
  or parent support is unavailable

**9. Communication System**
All communication runs through Google Classroom and WhatsApp. For each pair and each group define:
- What is communicated
- How often
- Which tool and which WhatsApp group
- Response time expectation
- What belongs in group message vs. direct message
- Escalation path

Cover: student–mentor, mentor–tutor (Group 2), mentor–parent (direct), tutor–parent (Group 3), program lead–tutors and mentors (Group 1), program lead–Gurudwara leadership (Group 1).

Also define WhatsApp norms for all four groups:
- Group 1 Leadership & Tutors: purpose, who posts, frequency, off-limits topics
- Group 2 Mentors & Tutors: purpose, daily flag format, weekly report format
- Group 3 Parent Communication: purpose, tone, frequency, bilingual
  requirement, what not to post
- Group 4 Students (if activated): purpose, moderation, boundaries

**10. Motivation and Recognition System**
For tutors, mentors, students, parents, and program lead include:
- Intrinsic motivators
- Community recognition
- Certificates and service hours
- Recommendation letters for mentors and tutors
- Student milestone celebrations
- Annual excellence event
- How Google Classroom progress data is made visible to motivate
  students and families

**11. Launch and Pilot Operations**
Practical 90-day launch plan:
- Pre-launch checklist: Google Classroom setup, four WhatsApp groups created, content curation workflow established, volunteer onboarding complete, student intake done, parent orientation complete (bilingual)
- Week-by-week operating rhythm for first 90 days
- Flipped classroom compliance tracking week by week
- First 2 weeks contingency plan: what to do if many students arrive repeatedly unprepared, including how to reset expectations without collapsing the pilot model
- Day 30 review: is the operating rhythm working?
- Day 60 review: are students progressing? Are mentors holding?
- Day 90 review: go / no-go for Phase 2; what to keep, change, or drop

**12. Risk Management**
How the manual handles each risk with a specific response protocol:
- Students not completing Google Classroom pre-work
- Content inconsistency between KA and test-prep sources
- Tutor burnout
- Mentor overload
- Student disengagement
- Parent disengagement
- WhatsApp group overload or misuse
- Role confusion between mentor and tutor
- Early pilot instability in Weeks 1–2, especially repeated
  pre-work non-compliance

**13. Continuous Improvement**
Simple monthly system for:
- Collecting feedback from all personas
- Reviewing flipped classroom compliance rates
- Improving content curation quality
- Improving mentoring effectiveness
- Making go / no-go decisions on Phase 2 subjects and expansion

**14. Templates and Tools**
- Mentor daily WhatsApp check-in log
- Tutor weekly report
- Parent weekly WhatsApp update in English and Punjabi
- Student weekly scorecard
- Google Classroom weekly content schedule template
- Content source selection log (source, topic, week, grade band)
- Attendance tracker
- Intervention tracker
- WhatsApp group message templates for common situations
  in English and Punjabi where parent-facing

**Output format:**
Clean, actionable handbook with checklists and role clarity throughout. Pilot phase first, expansion second. Plain language — written for community volunteers. Math-specific where relevant in the pilot phase, with notes on how English and Science will differ in Phase 2. Parent-facing materials and WhatsApp templates must be provided in both English and Punjabi.
