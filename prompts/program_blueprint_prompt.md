## MASTER PROMPT 1 — Program Blueprint (Final)

You are an expert in education systems design, community learning programs, mentoring systems, and high-performance academic pathways including PSAT, SAT, ACT, and National Merit.

Help me design a complete Gurudwara-based College Prep / Academic Excellence Program (CCP) for students in grades 7–11, using the configuration defined above.

The program should be designed as an academic acceleration system, not a generic tutoring program. Its purpose is to build strong mastery starting with Math, develop college entrance exam readiness progressively, and create a pipeline toward National Merit recognition, scholarship success, and student leadership.

**OVERARCHING DESIGN REQUIREMENT:**
Design the pilot as minimum viable and low-complexity first, then show how to expand. Every section must answer: what is the simplest version of this that still works? Complexity should be added only when the simpler version has been proven.

**IMPORTANT DESIGN PRINCIPLES:**
1. This is not homework help or catch-up tutoring. It is an acceleration and college prep system.
2. The pilot launches with Math only. English and Science are Phase 2.
3. The flipped classroom model is the core delivery mechanism. Students consume Google Classroom content before sessions. Tutor sessions are for application and clarification only.
4. Content curation is a distinct operational function — not an implicit tutor responsibility. Someone must own the weekly Google Classroom content workflow.
5. Flipped classroom compliance is the single greatest operational risk. The design must include pre-work verification, a non-compliance protocol, and mentor involvement in daily accountability.
6. Mentors are accountability partners only. They do not teach content. They verify practice, encourage students, and escalate issues.
7. WhatsApp has four defined groups as specified above. Communication norms must be explicit for each group.
8. The program must function even when parents are minimally available.
9. The program must avoid: weekly teaching without daily practice; tutors carrying all the load; no structured academic pathway; overdependence on parents; no culture of achievement.
10. The program is rooted in Sikh values: seva, discipline, humility, peer support, Charhdi Kala, Sarbat da Bhala.

**Please create a detailed program blueprint with the following sections:**

**1. Executive Vision**
- Why the program exists and why now
- Expected community impact
- Long-term student and family outcomes
- Alignment with Sikh values

**2. Program Goals**
- 90-day pilot goals (Math only, minimum viable)
- 1-year goals (Math proven, English and Science introduced)
- 3-year goals
- 5-year goals

**3. Core Program Components**
Define the minimum viable version and the expanded version of each:
- Flipped classroom delivery via Google Classroom
- Content curation and source hierarchy (primary, secondary, test-prep)
- Tutor-led weekly sessions
- Daily practice system
- Mentor / accountability-buddy system
- Peer teaching culture
- Parent engagement model
- PSAT / SAT / ACT pathway tied to the milestone timeline
- National Merit pathway
- Student recognition and achievement culture
- Progress tracking system
- Leadership and governance model

**4. Persona-Based Program Design**
For each persona — Program Lead, Content Curator, Tutors, Mentors, Parents, Students, Gurudwara leadership — provide:
- Mission of the role
- Responsibilities (pilot phase vs. expanded phase)
- Weekly and daily time commitment
- Reporting and communication lines including WhatsApp group assignment
- Decision authority
- Success metrics
- What could go wrong in this role
- How to keep this persona motivated over time

**5. Recruitment Plan**
For each persona provide:
- Who to recruit and where to find them
- Ideal profile and selection criteria
- Screening process
- Minimum viable numbers for pilot (use staffing assumptions from configuration block)

**6. Training Plan**
Create onboarding and training for Program Lead, Content Curator, Tutors, Mentors, and Parents. Include:
- Training topics per role
- Orientation agenda
- Flipped classroom model explanation for each persona
- Google Classroom usage expectations
- WhatsApp group norms and etiquette
- Mentor communication and escalation expectations
- Content source selection guidelines
- Keep training lean — volunteers have limited time

**7. Motivation and Retention Plan**
For each persona include both intrinsic and extrinsic motivators:
- Seva and purpose
- Community recognition
- Certificates and service hours
- Recommendation letters
- Leadership identity
- Visible student progress
- Milestone celebrations and annual recognition

**8. Student Progress Engine**
Design the daily and weekly operating system. Include:
- Daily 30–45 minute practice model via Google Classroom
- Mentor daily check-in routine with Google Classroom verification
- Pre-session preparation check by tutor
- Weekly review rhythm
- Attendance and commitment thresholds as defined in configuration
- Escalation ladder: missed practice → mentor flags → tutor reviews → program lead acts
- Minimum viable practice protocol for days when technology or parent support is unavailable
- How the system functions when parents are minimally engaged

**9. Academic Model — Math Pilot**
Design the Math-first academic framework. Include:
- How mastery sequencing works in Google Classroom
- How KA (primary) and Princeton Review or Kaplan (test-prep) are used together without creating conflicting explanations
- Content source selection protocol: how the Content Curator decides which source to use for which topic
- How PSAT/SAT Math reasoning is introduced progressively by grade band
- How advanced students are identified and stretched
- How peer teaching is built into the Sunday session model
- Phase 2 note: briefly describe how English and Science will be added once Math is proven, including their different instructional approaches

**10. Weekly and Monthly Operating Rhythm**
Describe the minimum viable operating rhythm:
- A typical student week (Google Classroom touchpoints, mentor check-in, Sunday session)
- A typical mentor week (WhatsApp Group 2 activity, daily check-ins, weekly report)
- A typical tutor week (content verification, session delivery, mentor coordination)
- Content Curator weekly workflow (source selection, upload schedule, format standards)
- Monthly review meeting agenda
- Parent communication rhythm via WhatsApp Group 3
- Leadership review cadence via WhatsApp Group 1

**11. 90-Day Pilot Plan**
Design a minimum viable 90-day launch. Include:
- Pre-launch checklist: Google Classroom setup, WhatsApp group creation, content curation workflow, volunteer onboarding, student intake, parent orientation
- Weeks 1–4: foundation (establish operating rhythm, flipped classroom compliance baseline)
- Weeks 5–8: acceleration (identify struggling students, introduce peer teaching)
- Weeks 9–12: review and adjustment (measure against 90-day goals, prepare Phase 2 recommendation)
- Flipped classroom compliance monitoring week by week
- Recommendation on whether grade bands should run together or separately during the pilot, with rationale based on minimum viable staffing
- Day 30, Day 60, Day 90 review checkpoints with specific questions to answer at each
- Success metrics for the pilot
- Go / no-go criteria for expanding to Phase 2

**12. Risks and Failure Prevention**
For each risk provide specific mitigation steps:
- Flipped classroom non-compliance
- Content inconsistency across sources
- Tutor burnout
- Mentor overload
- Student disengagement
- Parent disengagement
- WhatsApp noise and communication overload
- Mission drift into generic tutoring

Also explain explicitly how the design avoids the five structural failure modes:
- Weekly teaching without daily practice
- Tutors carrying all the load
- No clear academic pathway
- Parent support assumed
- No culture of achievement

**13. Organizational Model**
Show how all personas fit together operationally. Include:
- Org chart (text-based is fine)
- Communication flow across the four WhatsApp groups
- Accountability flow from student to mentor to tutor to program lead
- Decision-making authority at each level

**14. Scaling Plan**
Show the path from pilot to institution:
- Phase 1: Math pilot (10–15 students)
- Phase 2: Add English and Science; expand student intake
- Phase 3: Multi-grade, multi-tutor, mentor pipeline
- Phase 4: Student leaders become mentors; mentors become tutors
- Phase 5: Self-sustaining culture of excellence; Fremont replication

**15. Final Deliverables**
Provide:
- Concise program summary (one page)
- Launch checklist
- Org chart
- Pilot staffing plan using configuration block numbers
- Sample weekly schedule showing all Google Classroom and WhatsApp touchpoints
- Motivation summary by persona

**Output format:**
Write as a practical blueprint suitable for Gurudwara leadership. Be concrete and implementation-oriented. Always distinguish between pilot phase and expanded phase. Where relevant distinguish between grade bands (7–8, 9–10, 11). Use plain language — this will be read by community volunteers, not education professionals.
